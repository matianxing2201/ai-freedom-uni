## Why

当前“我的”页面仍是占位内容，既没有还原设计稿中的已登录态个人中心，也缺少未登录态的登录/注册入口，导致用户无法查看作品数据、进入服务入口或完成账号接入。现在需要将“我的”页面升级为一个完整的双态页面，既匹配设计稿视觉，也承接手机号登录、微信登录与注册流程。

## What Changes

- 重建 `src/pages/me/me.vue`，完整还原设计稿中的已登录态个人中心页面
- 新增未登录态界面，保持与个人中心统一的视觉语言，提供手机号登录、微信登录、用户注册入口
- 接入用户认证与用户信息接口：登录、发送验证码、查询用户信息、更新用户信息、绑定手机号/邮箱、发送绑定验证码
- 根据登录态在已登录界面与未登录界面之间切换，并在登录成功后刷新用户信息
- 使用 `src/static/images/mine` 目录下素材完成卡片、服务列表和箭头/图标展示

## Capabilities

### New Capabilities
- `me-page-states`: “我的”页面双态能力，覆盖已登录态个人中心 UI、未登录态引导 UI、登录态切换与页面交互
- `me-auth-entry`: “我的”页面认证入口能力，覆盖手机号验证码登录、微信登录、注册入口、登录成功后刷新用户信息
- `me-profile-actions`: 个人中心资料与服务入口能力，覆盖头像/昵称展示、我的作品统计卡片、账号设置/帮助反馈/关于我们入口，以及资料编辑与绑定接口承接

### Modified Capabilities
（无）

## Impact

- 修改 `src/pages/me/me.vue` 为完整页面实现
- 可能新增 `src/api/me.ts` 或 `src/api/account.ts` 以承接 4.1-5.5 的接口封装
- 可能新增“我的”页局部组件，如作品统计卡、服务列表项、登录弹层/表单组件
- 依赖现有 `src/store/token.ts`、`src/store/user.ts` 管理登录态与用户信息
- 使用 `src/static/images/mine` 目录下图标素材完成 UI 还原
